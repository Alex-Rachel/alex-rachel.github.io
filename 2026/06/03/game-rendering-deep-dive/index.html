游戏渲染技术全方位解析 - 从一根头发到一片云的技术全景

游戏渲染技术全方位解析 🎮✨

本文基于网易游戏学堂的技术分享，深入拆解游戏渲染的核心原理——从角色皮肤上的次表面散射，到天空中飘过的体积云，看技术美术如何用代码和数学在屏幕上重构一个真实可感的世界。

原文: 游戏渲染技术|全方位解析游戏画面的实现逻辑 — 网易游戏学堂

一、什么是游戏渲染？

游戏渲染（Game Rendering），简单来说就是让你能在显示器上看到游戏画面的技术。

从 2000 年的《Quake 2》到 2022 年的《地平线：西之绝境》，20 多年间游戏画面发生了翻天覆地的变化，但底层做的事情始终如一：通过计算机图形学算法，将 3D 场景转化为 2D 图像。

那些让人心动的画面，背后是一场关于”光”的精密计算。

二、渲染画面的三大核心模块

游戏渲染可以归纳为三个互相交织的核心模块：

模块 关注点 目标
角色（Character） 材质与交互：头发、皮肤、眼睛、服饰 让角色看起来真实或有风格
场景（Scene） LOD 与流送：植被、云、水、地形 构建可信的游戏世界
氛围（Atmosphere） 参数与控制：天气、光影（TOD 时间变化） 营造沉浸感

关键洞察：三者并非割裂，而是存在大量交集——玩家踩水产生波纹、风吹草动、场景破坏等交互效果，正是提升沉浸感的关键，也是技术实现的难点。

三、角色渲染技术详解

3.1 角色制作流程

从原画设计到游戏画面，角色需经历：

原画设计 → 3D 建模 → 材质制作 → 渲染输出

角色材质分为两大类：

  • 通用材质库：解决一般物体表面的光照问题
  • 特殊材质：眼球、皮肤、头发等需要特殊处理

3.2 PBR vs NPR：两大渲染范式

这是游戏渲染中最核心的概念分野：

特性 PBR NPR
驱动方式 公式驱动（Formula Driven） 结果驱动（Result Driven）
追求目标 物理正确性 艺术风格表达
代表类型 写实类游戏 二次元 / 风格化游戏
美术工作流 标准化，工具成熟 需自定义，成本较高

3.3 渲染方程

所有基于物理的渲染都建立在这个方程之上：

它描述了从表面某点沿观察方向 ωo 射出的光量，等于自发光加上所有入射光经过 BRDF（双向反射分布函数）调制后的积分。

3.4 PBR 的材质基础：从 MERL 到 Disney Principled BRDF

在 PBR 流行之前，研究者用专业设备测量真实材质的反射特性，建立了 MERL BRDF 数据库。但真实数据驱动存在瓶颈：

  • 数据量大（33MB+），搜索性能低
  • 无法表达次表面散射、各向异性等
  • 材质范围受限于采样

Disney Principled BRDF（2012） 带来了革命性的简化：

  1. 参数直观：如”粗糙度”而非晦涩的物理参数
  2. 参数尽可能少：控制在少数几个关键参数
  3. 值范围 0-1：标准化，便于理解和调整
  4. 允许参数超出范围：在有意义时突破限制
  5. 组合健壮：任意参数组合不产生错误

核心参数：粗糙度（Roughness）/ 光泽度（Glossiness），两者为相反关系。这一原则彻底改变了游戏美术的工作流。

3.5 微表面模型（Microfacet Model）

当像素覆盖大量表面细节时，基于统计学描述表面的微小法线变化。

BRDF 的两种成分：

  • 漫反射（Diffuse）：一般简化为 1/π
  • 镜面反射（Specular）：使用 GGX 模型 + Cook-Torrance BRDF

多层材质混合的演进：

  • 传统方式：基于结果的混合（物理上有问题）
  • 更优方案：材质计算阶段提前混合（线性叠加）
  • UE5 的 Strata 系统：用”slab”概念表达每层的光学结构，实现更准确的线性混合

3.6 NPR（非真实感渲染）风格化

二次元渲染的核心原则：所见即所得。

  • 贴图即结果：美术画的贴图直接对应最终效果
  • 光照烘焙：光照信息预烘焙到贴图中
  • 边缘光：利用菲涅尔（Fresnel）系数实现
  • 屏幕空间处理：在屏幕空间做卷积得到更细的边缘光

3.7 皮肤渲染（Skin Rendering）

皮肤与通用表面材质最大的不同：多层结构（表皮、真皮、油脂、血管）带来的次表面散射（SSS）。

核心现象：

  • 正面通透感：亮部到暗部的柔和过渡
  • 背面透射：光从背面穿透薄皮肤
  • 细节高光：更细、更丰富的高光层次

SSS 是让皮肤看起来”活”的关键——没有它，皮肤就像塑料。

3.8 头发渲染（Hair Rendering）

头发的特殊性：圆柱形纤维，非平面。光路行为有三种：直接反射、同侧透射、穿透散射。

光照模型：

  • Kajiya-Kay 模型（1989）：基于切线计算高光，公式简单，手游主流
  • Marschner 模型（2003）：完整解决三条光路，使用预计算 LUT 加速

建模方案上，从卡片（Card）到发丝级（Strand）渲染，在效果和性能之间做取舍。

3.9 眼球渲染（Eye Rendering）

关键在于结构拆解：

  • 角膜：折射、反射
  • 虹膜：散射、纹理
  • 巩膜：血管纹理、SSS
  • 泪膜：高光反射

最新趋势包括动态效果：肌肉跟随、Look At 功能的细节优化。

角色渲染小结

核心原则：用更低成本达到更好效果。 PBR 方案因标准化工作流而成本更低；NPR 方案需自定义工具链，成本较高。

四、场景渲染技术详解

4.1 场景制作的 PCG 流程

现代游戏场景制作大量使用程序化内容生成（PCG）：

步骤 工具/技术 输出
地形生成 高度图雕刻 基础地貌
河流切割 程序化工具 河道
悬崖细节 Houdini 等 岩石细节
植被布置 生态刷子 树木、草地
最终整合 场景编辑器 完整场景

4.2 地形渲染（Terrain Rendering）

何时需要地形系统？

场景规模 推荐方案
< 200 米 普通网格模型
200 米 - 1 公里 视情况选择
> 1 公里 地形系统（推荐）
1000+ 平方公里 大世界地形系统

核心技术：

  • 高度图（Heightmap）：每米/每像素存储一个高度值，实时生成网格
  • LOD（Level of Detail）：根据距离调整网格密度
  • Clipmap / 四叉树：高效管理大区域地形数据
  • VT（Virtual Texture）：虚拟纹理优化材质混合性能
  • GPU Driven：GPU 直接生成网格，减少 CPU 开销

地形材质混合：使用权重图混合多种材质（草地、岩不、泥土等），通过只保留权重最高的两层材质来优化性能。

4.3 植被系统（Vegetation System）

核心目标：在有限性能预算内渲染足够密度的植被。

创作方式：

  • 密度图刷：用刷子绘制密度，运行时按阈值生成（大范围植被）
  • 精确撒点：手动指定每个植被的位置（精确控制区域）

植被交互——通过风场（Wind Field）和物理模拟实现：

  • 风吹摆动：牛顿力学 F = ma
  • 角色碰撞：角色附近绑”风力源”
  • 实时响应：与环境和角色的动态交互

4.4 水体渲染（Water Rendering）

水体光照成分：

  • 反射（Reflection）：SSR 或光追
  • 折射（Refraction）：水下透视效果
  • 焦散（Caustics）：光聚焦形成的光斑
  • 次表面散射（SSS）：水的通透感
  • 深浅颜色渐变：浅水/深水不同颜色

水体网格：使用 CDLOD（Continuous Distance LOD） 实现平滑过渡——重叠区域两套 LOD 网格共存，根据距离做渐变混合。

4.5 天气系统：天空、大气、云、雾

统一的技术本质：Ray Marching（光线步进）——所有介质的光照问题都可以通过这一核心算法求解。

物理大气（Atmosphere）

  • 经典论文：Preetham 2008《A Practical Analytic Model for Daylight》
  • 建模：两层壳模型（地球内核 + 外层大气）
  • 预计算 LUT：根据高度、视角、光源角查表
  • 散射类型：瑞利散射（蓝天）+ 米氏散射（雾天）
  • 多次散射：迭代 LUT 实现间接光照

大气雾（Aerial Perspective）

解决传统高度雾与大气渲染割裂的问题：使用深度切片（32×32×16）融入大气计算，通过透射率/散射亮度实现无缝衔接。

体积云（Volumetric Cloud）

效果最逼真但性能开销最大。使用 3D 密度场 + Ray Marching，云层可遮挡光照影响场景。

云的层级化方案：

类型 实现方式 性能
雾（低层云） 体积雾类似方案 低
普通云层 Billboard/Impostor 中
体积云 Ray Marching 密度场 高

场景渲染小结

场景制作的三大挑战：

挑战 说明
内容 创作工具（笔刷、PCG、AIGC）决定生产效率
性能 LOD、剔除、流送、GPU Driven
效果 风/水交互、场景破坏、UGC 挖洞等

五、总结

渲染技术选择原则

场景 推荐方案
写实类 3A 游戏 PBR + 物理准确方案
移动端游戏 简化 PBR + LUT 预计算
二次元/风格化 NPR + 所见即所得
混合风格 分层材质系统（如 UE5 Strata）

性能优化通用套路

  1. LOD 策略：距离越远，细节越少
  2. 剔除优化：视锥剔除 + 遮挡剔除
  3. 流送管理：只加载需要的数据
  4. GPU Driven：让 GPU 承担更多工作
  5. 预计算 LUT：空间换时间

核心学习路径

  • 基础：渲染方程 → BRDF → 微表面模型
  • 进阶：Disney PBR → SSS → 各向异性
  • 场景：地形 LOD → 植被渲染 → 水体模拟
  • 氛围：大气散射 → Ray Marching → 体积云

写在最后

“渲染的本质是用最低的成本，达到最好的效果。”

游戏渲染是一个效果、性能、成本三者平衡的艺术。没有银弹方案，只有最适合当前项目的选择。理解底层原理，掌握工具链，才能在实际开发中做出正确的技术决策。

从一根头发的光线追踪，到整片天空的大气散射——每一个像素背后，都是数学和工程的精密配合。这正是游戏渲染技术的魅力所在。
